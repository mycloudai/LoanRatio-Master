## [0.6.2] - 2026-04-21

### Fixed

- **手动月份权益比例显示不一致（关键修复）**：历史月份表格及汇总历史中，手动模式下显示的「比例」列原来是当月手动设定的还款比例，与自动模式（显示 `CP_i/CP` 权益比例）定义不一致。现统一为：所有历史月份（自动/手动）的「比例」均为该月本金模型更新后的累计权益比例 `CP_i(t)/ΣCP(t)`，语义一致。
- **手动月份详情弹窗缺少权益比例列**：手动月份详情弹窗新增「截止到本月的所有权比例」列，与自动模式保持一致显示。

## [0.6.1] - 2025-07-23

### Fixed

- **预测回测比例泄漏（关键修复）**：修复两处导致 0% 比例参还人在预测中错误获得份额的 bug
  - `calculator.py`：负本金重分配回退逻辑中，当 S+ 参还人历史比例为 0 时（`denom ≤ 0`），不再将赤字均分给他们，改为仅保留自身贡献
  - `main.py`：预测平均还款计算中，手动月份现按 `manualRatios` 加权归属还款额，而非使用原始 `payerPayments`（例如手动 100%/0% 的月份中，0% 参还人的实际银行转账不再被计为自身贡献）
- **汇总权益比例显示错误**：汇总页和历史快照的权益比例现始终基于累计本金计算（`CP/ΣCP`），不再使用手动月份的覆盖比例，避免误导

### Changed

- **汇总标题优化**：「各参还人当前权益」改为「各参还人最新累计权益」，明确表示数据为所有历史月份的累计结果；历史快照标题显示截至月份

### Added

- **Sanity 综合计算验证测试**：QA 新增第 54 节，含 81 项断言覆盖 3 人 2 贷 9 月混合场景及 100%/0% 边界情况
- **`--sanity-only` 模式**：`bash qa/run_qa.sh --sanity-only` 可仅运行 sanity 测试，适用于未涉及功能新增的变更

### Improved

- **AI 指令更新**：新增 sanity 测试文档和测试选择指南表

## [0.6.0] - 2026-04-21

### Added

- **各贷款还清预测**：预测端点新增 `loanForecasts` 数组，逐笔贷款计算剩余本金、还清月数、预计还清月份、总未来利息（替代旧的 `estimatedRemainingInterest` 粗估值）
- **内联 SVG 图表**：权益趋势折线图和贷款利息柱状图使用纯 SVG 渲染，移除对 Chart.js CDN 的外部依赖
- **历史快照下拉选择器**：汇总页底部新增下拉选择器，选择月份后以只读模式查看该时间点的权益状态
- **确认弹窗 Bootstrap 模态框**：还款金额不一致提示改为 Bootstrap 风格模态框，与系统其他提示风格统一

### Changed

- **预测排除首付月**：首付月（0000-00）不再参与预测平均值计算
- **预测标签优化**："预测时长（月）"改为"向未来预测月数"，语义更清晰
- **编辑弹窗手动比例可见性**：自动模式下编辑月份时隐藏手动比例输入区域，切换到手动模式时自动显示

### Fixed

- **fmtCurrency 不存在**：修复预测结果显示中使用不存在的 `fmtCurrency` 函数导致报错的问题（已随整体重写移除）
- **ruff lint 错误**：修复 `tests/test_api.py` 中 4 个 ruff 检查错误（B905 zip strict、E741 变量名、B007 未使用变量）

### Improved

- **USERGUIDE 更新**：预测章节和汇总章节更新，新增还款不一致提示说明和历史快照使用说明

## [0.5.0] - 2026-04-21

### Added

- **还款金额不一致提示**：新建或编辑月份时，若银行应还款总额（利息+本金）与参还人实际还款总额不一致，弹出确认对话框提示用户核实（确认后仍按实际填写保存）
- **手动月份计算详情**：手动比例月份的详情弹窗现在显示各参还人还款额、手动比例、实际本金推导过程（`max(0, 还款总额 − 利息) × 比例`），不再显示自动模式的四步公式
- **手动月份详情表格优化**：手动月份详情表格列调整为"还款额 / 手动比例 / 计入本金 / 累计本金"，更符合手动模式语义
- **新增单元测试**：手动月份详情步骤内容校验
- **新增 QA 测试**：还款不一致确认对话框、手动月份详情步骤内容

## [0.4.0] - 2026-04-21

### Changed

- **手动模式本金计算**：手动模式下累计本金改为按 `手动比例 × max(0, 参还人还款总额 - 利息)` 计算，与自动模式逻辑保持一致；若参还人还款总额不足以覆盖利息则本金贡献为零

### Added

- **预测剩余利息**：预测端点新增 `estimatedRemainingInterest` 字段，估算贷款剩余期间总利息
- **预测图表数据**：预测端点新增 `series`（各参还人比例时间序列）和 `months`（月份标签数组），修复前端趋势图无法渲染的问题
- **无数据预测校验**：未录入任何月份时调用预测端点返回 400 错误，而非静默返回空结果
- **手动模式欠付测试**：新增单元测试和 E2E 测试，覆盖手动模式下还款不足利息时本金归零的场景

## [0.2.1] - 2026-04-21

### Fixed

## [0.1.1] - 2026-04-21

### Fixed

- **预测端点浅引用污染**：forecast API 调用后不再污染真实 state 的 `remainingPrincipal`（浅拷贝 → 深拷贝贷款对象）
- **全员欠款场景**：当所有参还人还款不足以覆盖利息时，调整后本金归零，防止累计本金被侵蚀为负数
- **多贷款预测**：forecast 端点按每笔贷款分别计算平均利息/本金，修复多贷款场景预测不准确的问题
- **QA 脚本清理**：退出时清理通过端口号查找的残留服务器进程，防止孤儿进程

### Changed

- **QA 用例 23 更新**：全员欠款场景期望值更新以匹配新的归零逻辑（adj=0，CP 保持首付值）

### Improved

- **UI 视觉升级**：新增 Bootstrap Icons、渐变页头、改进卡片阴影与圆角、更好的表格样式与悬停效果、标签页图标、优化空状态提示

## [0.1.0] - 2026-04-21

### Added

- **核心计算引擎**：本金归因模型四步算法（利息分摊 → 原始净本金 → 负本金再分配 → 权益比例更新）
- **首付与零首付支持**：首付按出资比例建立初始权益；零首付首月按 1/n 平摊利息
- **自动 / 手动双模式**：手动模式直接设定比例且不更新累计本金
- **历史月份级联重算**：修改任意历史月份后后续所有月份自动重算
- **参还人管理**：增删改；支持起始月份；删除支持合并 / 直接删除两种策略
- **贷款管理**：增删改；支持原始金额与剩余本金；删除支持合并到其他贷款
- **汇总面板**：展示各参还人累计本金、当前权益、各贷款剩余本金、月度历史表格
- **Excel 导出**：含原生公式，用户可在 Excel 中直接修改数字
- **权益趋势预测**：基于近 N 月平均值预测未来月权益与贷款结清月份
- **版本 / 仓库 / CHANGELOG UI**：页头徽章、仓库链接、内嵌渲染的更新日志模态框
- **本地启动脚本**：`start.sh` / `start.bat` 一键启动
- **JSON 文件存储**：用户自选数据路径，原子写入
- **单元测试套件**：pytest 覆盖 calculator + API，38 用例
- **端到端 QA 脚本**：`qa/run_qa.sh` 覆盖全部核心场景，支持有头 / 无头模式
- **GitHub Actions**：`ci.yml` 单元测试 + lint；`qa.yml` main 推送触发 E2E 测试
- **开源项目规范**：MIT License、README、CONTRIBUTING、CODE_OF_CONDUCT、Issue/PR 模板
